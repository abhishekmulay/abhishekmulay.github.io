Do Not Forget Your New Keyword

May 3rd, 2015 4:04 pm | Comments

Missing new while creaing an object in javascript is a very common and a lethal bug.

What happens if you forget new while creating an Object ?

Let’s figure it out, shall we?

Let’s start from the basics. Suppose I have an Animal constructor and I want to make objets using constructor, I will do something like this

function Animal(){
    'use strict';
    this.color = 'Black';
}

var dog = new Animal();
console.log(dog.color); // Black

Now, let’s see what is really happening behind the scene when we create the dog object using new

1
2
3

9

function Animal() {
    'use strict';
    var this = {}; // this is not valid, just for illustration
    this.__proto__ = Animal.prototype;

    this.color = 'Black';

    return this;
}
  1. Here the this refers to the object of type Animal
  2. It sets the prototype of newly created object to Animal
  3. A new property is created on Animal object which has name color
  4. The current object this is returned.

Lets see what happens if we miss new and create the object like 

1
2
3
4

var cat = Animal();

console.log(cat); // undefined
console.log(cat.color); // TypeError: Cannot read property 'color' of undefined

But why ?

The reason is, when we invoke any Constructor function without the new keyword, the this reference within the constructor function will be undefined, hence any operation on undefined will throw a TypeError

This is the scenario when our code is running in strict mode. Read more about strict mode.

If the code is not running in strict mode, then the this reference inside the Constructor function points to the global window object. This makes all the new object properties global (i.e. the property of the window object) which is a VERY VERY bad thing!

/*This is not in strict mode*/
function Animal(){
    this.color = 'Black';
}

var cat = Animal();
console.log(window.color); // Black

So fellas, when creating a new object don’t forget your new keyword!

Happy javascripting!
