‘This’ Is Sparta : Understanding ‘Context’ in Javascript

Nov 24th, 2014 5:42 pm | Comments

Javascript has come a long way from being a language used for showing popups in a webpage to being the highest trending language on Github. Javascript is sexy but it might bite you if you are not aware of the intricacies of the language. For a javascript developer having a good grasp on the basics of the language is very important. One of the most important and misunderstood concepts in javascript is this or context.

So, without further ado, lets dive deep and understad what is the execution context.

In on senetence :

context in javascript is the value of this keyword.

If you are insecure and only trust the official word, here is the definition from official ECMAScript specification.

The this keyword evaluates to the value of the ThisBinding of the current execution context.

11.1.1 The this keyword - ECMAScript Language Specification

Okay now for people coming from Object Oriented background (java/C# etc) this might sound weird but in wonderful world of javascript the this can have different values depending upon the scope in which it was accessed. In most cases, the value of this is determined by how a function is called.

There are three possible execution contexts

  • Global context
  • Function context
    • Method of an object
    • As a constructor
    • call and apply
    • bind
  • A DOM Event handler

Global context

By default if you access this (outside of any function), then it refers to global object , in browserland it is the window object.

1

console.log(this); // Window {top: Window, window: Window, location: Location, external: Object, chrome: Object…}

Any variable written in the global scope becomes a global object. 

1
2

name  = "Jules Winfield";
console.log(Window.name);  // "Jules Winfield"

This is one of the worst parts of the language. Closures are used to save the souls which have wondered down the perilous path of using global variables.

Function context

Inside a function, the value of this depends on how the function is called. Let’s see what this means 

1
2
3
4
5

function callMe(){
    console.log('this = ',this); 
}

callMe();  //  this = window.

As we are not using strict mode, javascript returns an object the global window object. Let’s see what happens if we use strict mode.

1
2
3
4
5
6

function callMeStrictly(){
    "use strict";
    console.log('this = ',this); 
}

callMeStrictly();  //  this = undefined.

This is because in strict mode, the value of this remains at whatever it’s set to when entering the execution context hence the undefined in our case.

Whenever a method is a part of an object then it’s this reference points to that object. It does not matter if that method is member of that object or coming from it’s prototype, the this will always point to the same object.

var walter = Object.create(breakingBad);
walter.name = "Walter White";
walter.sayMyName(); // Walter White.

Here the this points to walter object because sayMyName method was called on that object and it does not matter that the method is not the member of walter object and it is inherited from its prototype. Object.create() method creates a new object with the specified prototype object and properties.

As a constructor

When calling any function with new, its this will point to the newly created object.

The handsome twins : Call and Apply

I call them handsome twins because their working is exactly similar, the only difference is call()accepts an argument list, while apply() accepts a single array of arguments.

(function(){
    'use strict';

    function Person(name){
        this.name = name;
    };

    var abhi = new Person('Abhishek');

    function whoAreYou(language){
        console.log(this.name +" is a " + language+" programmer.");
    }

    console.log(whoAreYou.call(abhi,'javascript')); // Abhishek is a javascript programmer.
    
})();

Here function whoAreYou is reading the name property off the this,but as we are in strict mode currently its this is undefined, then how does this work? What call does is it calls whoAreYou with its this set to whaterver is the first arguement to the call function, in our case that is the abhi object which has its own name property.

The evil twin bind

The Function.bind does exactly what call does, it changes the this reference of the function, but as he is an evil twin it doesn’t let go off the this even after the function being executed.

The bind() method creates a new function that, when called, has its this keyword set to the provided value, with a given sequence of arguments preceding any provided when the new function is called.

    boundSayMyName(); // Walter White

This code is running in the global context, hence the this is pointing to global object window. But inside the DOM click event handler the this keyword refers to the object to which we are attaching the event.
