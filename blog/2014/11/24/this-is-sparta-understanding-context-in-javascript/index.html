‘This’ Is Sparta : Understanding ‘Context’ in Javascript

Nov 24th, 2014 5:42 pm | Comments

Javascript has came a long way from being a language used for changing the colors after clicking a button in a webpage to being the highest trending language on Github. Javascript is sexy but it might bite you if you are not aware of the intricacies of the language. For a javascript developer having a good grasp on the basics of the language is very important. One of the most important and misunderstood concepts in javascript is context.

So, without further ado, lets dive deep and understad what is the execution context.

In on senetence :

context in javascript is the value of this keyword.

If you are insecure and only trust the official word, here is the definition from official ECMAScript specification.

The this keyword evaluates to the value of the ThisBinding of the current execution context.

11.1.1 The this keyword - ECMAScript Language Specification

Okay now for many of the people coming from java/C# like background this might sound weird but in wonderful world of javascript the this can have different values depending upon the scope in which it was accessed. In most cases, the value of this is determined by how a function is called.

There are 4 possible execution contexts

Global context

By default if you access this (outside of any function), then it refers to global object , in browserland it is the window object.

1

console.log(this); // Window {top: Window, window: Window, location: Location, external: Object, chrome: Object…}

Any variable written in the global scope becomes a global object. 

1
2

name  = "Jules Winfield";
console.log(Window.name);  // "Jules Winfield"

This is one of the worst parts of the language. Closures are used to save the souls which have wondered down the perilous path of using global variables.

Function context

Inside a function, the value of this depends on how the function is called. Let’s see what this means 

1
2
3
4
5

function callMe(){
    console.log('this = ',this); 
}

callMe();  // outputs this = window.

As we are not using strict mode, javascript returns an object the global window object. Let’s see what happens if we use strict mode.

1
2
3
4
5
6

function callMeStrictly(){
    "use strict";
    console.log('this = ',this); 
}

callMeStrictly();  // outputs this = undefined.

This is because in strict mode, the value of this remains at whatever it’s set to when entering the execution context hence the undefined in our case.
